## File Contents — What Each File Actually Does

### `prisma/schema.prisma`
The entire database schema. Defines 6 models and 2 enums.

**Critical Prisma 7 note**: The `datasource db` block has NO `url` field — that moved to `prisma.config.ts`. Do not add `url = env(...)` here, it will break.

```
datasource db { provider = "postgresql" }   ← no url here — Prisma 7 change
generator client { provider = "prisma-client-js" }
```

Models (see full schema section below).

---

### `prisma.config.ts`
**Prisma 7 only** — this file didn't exist in Prisma v6.
Tells Prisma CLI where the schema is and what DB to connect to.
```ts
import "dotenv/config";              // loads .env.local before reading env vars
import { defineConfig } from "prisma/config";
export default defineConfig({
  schema: "prisma/schema.prisma",
  datasource: { url: process.env.DATABASE_URL! },
});
```
When running `prisma db push`, dotenv reads `.env.local` → `DATABASE_URL` is set → Prisma connects to Neon.

---

### `src/lib/db.ts`
Prisma client singleton. The only place `PrismaClient` is instantiated.

**Why singleton**: Next.js dev server hot-reloads on every file save. Without the singleton, each reload creates a new `PrismaClient`, exhausting the free-tier Neon connection pool.

**Why `PrismaPg` adapter**: Prisma 7 no longer accepts raw connection strings in `PrismaClient({ datasources })`. A driver adapter is now required.

```ts
import { PrismaPg } from "@prisma/adapter-pg";
const adapter = new PrismaPg({ connectionString: process.env.DATABASE_URL! });
export const db = globalForPrisma.prisma ?? new PrismaClient({ adapter });
```

Import this as `import { db } from "@/lib/db"` in any server file.

## Database Schema — Full Explanation

All tables are live in Neon PostgreSQL (`neondb` database, `public` schema).

### `User`
```
id            String    @id @default(cuid())
name          String?
email         String    @unique
emailVerified DateTime?
image         String?
createdAt     DateTime  @default(now())
updatedAt     DateTime  @updatedAt
```
Owns: `accounts[]`, `courses[]`

NextAuth creates/updates this row on first Google login. `email` is unique — no duplicates if the same Google account signs in twice.

---

### `Account`
```
id                String   @id
userId            String   → User.id (CASCADE delete)
type              String   (always "oauth" for Google)
provider          String   (always "google")
providerAccountId String   (Google's user ID)
refresh_token     String?  @db.Text
access_token      String?  @db.Text
expires_at        Int?
token_type        String?
scope             String?
id_token          String?  @db.Text
session_state     String?
@@unique([provider, providerAccountId])
```
Managed entirely by `@auth/prisma-adapter`. We never write to this manually. Stores the Google OAuth tokens. Deleted when User is deleted (`onDelete: Cascade`).

---

### `Course`
```
id                String   @id @default(cuid())
userId            String   → User.id (CASCADE delete)
youtubePlaylistId String   (e.g. "PLxxxxxxxxx")
title             String
thumbnailUrl      String?
createdAt         DateTime
updatedAt         DateTime
@@unique([userId, youtubePlaylistId])
```
One imported YouTube playlist = one Course. A user can only import the same playlist once (enforced by the unique constraint). Owns `videos[]`.

---

### `Video`
```
id                 String      @id @default(cuid())
courseId           String      → Course.id (CASCADE delete)
youtubeVideoId     String      (e.g. "dQw4w9WgXcQ")
title              String
position           Int         (0-indexed order in playlist)
durationSeconds    Int?
thumbnailUrl       String?
isAvailable        Boolean     @default(true)
status             VideoStatus @default(not_started)
lastWatchedSeconds Int?
@@unique([courseId, youtubeVideoId])
@@index([youtubeVideoId])
```
One playlist item = one Video row. **Per-user watch state lives directly here** (status, lastWatchedSeconds) — no join table needed because Video already belongs to exactly one user via `Course → User`.

`isAvailable=false` → YouTube video was private/deleted at import time. Row is kept; embed will show "Video not available" modal.

`@@index([youtubeVideoId])` — enables fast lookup when checking `AIContent` for a video.

Owns `notes[]`.

---

### `AIContent`
```
id             String          @id @default(cuid())
youtubeVideoId String          @unique   ← THE dedup key
transcript     String?         @db.Text
summary        String?         @db.Text
quiz           Json?
status         AIContentStatus @default(pending)
createdAt      DateTime
updatedAt      DateTime
```
**Shared across ALL users** — keyed by YouTube video ID, not by our internal Video.id or by userId. If User A and User B both import the same YouTube video, they share one `AIContent` row.

The `@unique` on `youtubeVideoId` is the dedup lock:
- First request: INSERT → succeeds → generates AI content
- Concurrent second request: INSERT → unique violation → catches error → reads existing row

`quiz` is a JSON column with shape:
```json
{
  "questions": [
    { "question": "...", "options": ["A", "B", "C", "D"], "answer": 0 }
  ]
}
```
`answer` is the 0-indexed position of the correct option.

---

### `Note`
```
id        String   @id @default(cuid())
videoId   String   → Video.id (CASCADE delete)
content   String   @db.Text
createdAt DateTime
updatedAt DateTime
```
Free-text notes per video. No `userId` field — ownership is implied via `Note → Video → Course → User`. When checking auth in API routes, verify `Note.video.course.userId === session.user.id`.

---

### Enums
```
VideoStatus:    not_started | watching | completed
AIContentStatus: pending | ready | failed
```

## Current Architecture Flow

This is how the code connects RIGHT NOW (not future state):

```
HTTP Request (browser)
        ↓
next.config.ts          ← Next.js reads this first (currently empty/no rules)
        ↓
src/app/layout.tsx      ← Root layout wraps every response
  - Loads Geist fonts
  - Sets <html> + <body>
        ↓
src/app/page.tsx        ← Handles GET / (boilerplate page currently)
        ↓
src/app/globals.css     ← Tailwind v4 styles applied globally
```

**Database connection flow** (when any server component or API route needs the DB):
```
import { db } from "@/lib/db"
        ↓
src/lib/db.ts
  - Checks globalThis.prisma (singleton check)
  - new PrismaPg({ connectionString: process.env.DATABASE_URL })
  - new PrismaClient({ adapter })
        ↓
Neon PostgreSQL (via connection pooler URL in .env.local)
  ↓ returns typed query results based on prisma/schema.prisma
```

**Environment variable flow**:
```
.env.local                         ← read by Next.js automatically at dev time
  DATABASE_URL → src/lib/db.ts     ← PrismaPg connection string
  GOOGLE_CLIENT_ID →               ← used in Step 2 (NextAuth config, not yet created)
  GOOGLE_CLIENT_SECRET →           ← used in Step 2
  NEXTAUTH_SECRET →                ← used in Step 2
  YOUTUBE_API_KEY →                ← used in Step 3
  GEMINI_API_KEY →                 ← used in Step 7

prisma.config.ts                   ← read by Prisma CLI only (not Next.js runtime)
  uses dotenv to load .env.local
  DATABASE_URL → prisma db push / prisma generate
```

## Key Patterns to Follow

### Importing the DB client
```ts
// Always import from @/lib/db — never create a new PrismaClient anywhere else
import { db } from "@/lib/db";
```

### Route handler signature (Next.js 15 — params is a Promise)
```ts
// CORRECT — params must be awaited in Next.js 15
export async function GET(
  request: Request,
  { params }: { params: Promise<{ courseId: string }> }
) {
  const { courseId } = await params;
}
```

### Ownership check pattern (used in every API route)
```ts
// Never trust the ID from the URL alone — always verify via join
const video = await db.video.findFirst({
  where: { id: videoId, course: { userId: session.user.id } }
});
if (!video) return Response.json({ error: "Not found" }, { status: 404 });
```

### Running Prisma CLI commands
```bash
npx prisma db push       # push schema changes to Neon (reads prisma.config.ts → .env.local)
npx prisma generate      # regenerate client after schema changes (Prisma 7: NOT automatic)
npx prisma studio        # visual DB browser
```
